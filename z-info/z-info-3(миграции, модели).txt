1. В database\migrations создаём новую миграцию: php artisan make:migration create_products_table

2. К автоматически созданным полям добавляем необходимые нам поля будущей таблицы

3. Чтобы выполнить все невыполненные миграции(создать соотетствующие таблицы в БД), выполните команду: php artisan migrate

-Откатить миграции: php artisan migrate:rollback (при этом все имеющиеся таблицы с данными в них будут удалены из БД) Затем можно изменять имеющиеся миграции(таблицы) После внесения изменений, нужно опять запустить миграции: php artisan migrate

-Откатить миграции (например на 1-н шаг): php artisan migrate:rollback --step=1 Будет удалена из БД только последняя созданная миграция(таблица)
Далее действия аналогичные описанным выше: вносятся необходимые изменения в миграцию и запускаются миграции

-Внести правки в существующую миграцию(таблицу): php artisan make:migration update_products_table
После внесения изменений, нужно опять запустить миграции: php artisan migrate

-Создание нового поля в уже имеющейся таблице (ч.11: Создание Middleware, Auth):
	$php artisan make:migration alter_table_users  В миграции прописать необходимое поле и запустить миграцию

- тоже самое, но появившаяся миграция с указанием названия изменяемой таблицы (ч.17: Checkbox, Mutator):
	$php artisan make:migration alter_table_products --table=products

4. Создадим модель в app\Models которая будет работать с соответствующей таблицей БД (Называем её как и таблицу,
но с большой буквы и в единственном числе): php artisan make:model Product

Чтобы создать одновременно модель и соответствующую ей миграцию:  php artisan make:model -m Category
